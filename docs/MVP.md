# MVP Camino

El primer objetivo no es construir todos los modulos posibles. El primer objetivo es que una persona entre y sienta: "esto me hizo sentir acompanada".

## MVP Inicial

1. Landing hermosa
2. Login y registro
3. Onboarding emocional
4. Dashboard calido
5. Chat IA
6. Diario
7. Reflexion diaria
8. Modo ansiedad

## Criterio de Producto

El MVP debe:

- Sentirse real, no como demo vacia
- Usar datos reales cuando una fase lo requiera
- Tener copy humano y sobrio
- Mostrar disclaimers de seguridad
- Recomendar ayuda humana cuando corresponda
- Ser facil de probar y mejorar

## Rol del Director de Producto

El desarrollo por fases necesita validacion humana continua:

- Probar cada flujo
- Corregir UX
- Decidir prioridades
- Cuidar identidad
- Mantener foco emocional
- Evitar construir modulos sin aprendizaje real
